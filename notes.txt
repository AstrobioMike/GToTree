### ADDED ###

# --wanted-ref-tax parameter added to main `gtotree` program
    # this allows specifying a taxon at any rank in order to make a tree of genomes from that taxon
    # this is so the user no longer needs to collect their own accession input lists anymore
# added a gene-represenation cutoff (-g) parameter
    # this is the minimum proportion of genomes required to have hits to a target gene for that target gene to be retained for downstream processing
    # e.g., with 100 input genomes and a -g of the default 0.2, any target gene that has hits in less than 20 genomes will be filtered out
    # This is calculated based on the total number of genomes remaining
    #   after preprocessing steps, but before genome filtering based on the '-G'
    #   parameter, and it is not revisited after genome-level filtering (so as to
    #   avoid iterative pruning effects between the '-g' and '-G' parameters)
# --resume flag added to attempt to continue interrupted runs
# helper programs are now subcommands under `gtt`
    # run `gtt` by itself or `gtt -h` for an overview

# lots of things since initial v1 release, but were integrated before v2

### CHANGED ###

# command callable with lowercase `gtotree` or original `GToTree` now (i don't know what i was thinking almost a decade ago)
# the code-base has been virtually entirely rewritten from
# when calculating percent of target genes required to be found (-X parameter)
    # now it uses the amount remaining after filtering out target genes with too few genomes having hits as the denominator
    # previously it used the starting total as the denominator
        # example: if starting with 100 target genes, and 20 are filtered out for having hits in too few genomes, then the percent required is based on 80 target-genes now rather than 100
# default number of jobs set to 4 instead of 1 (-j parameter)
# default output changed to "gtotree-output"
# dropped '-n' num HMM cpus argument (for scanning individual genomes, the default of 2 is virtually always fine)
# default lineage settings are D,P,C,S (no longer strain)
# printed output has more progress bars and fewer text printouts
# better logging of many steps
# http is used instead of ftp always where available (used to be ftp by default with an http option)
# output files and dirs that had underscores have had them replaced with dashes (sorry, i now hate underscores when they're not necessary)
# estimated percent completion and redundancy are no long provided
    # (this was always just "free" info, but checkm2 is a much better way to do this, so i made the decision to stop providing it at all)
# taxids are no longer searched for in input genbank files, they will only be found/used for provided input ncbi-accessions
    # due to the nature of genbank files, this was always a little messy and hacky. If this is actually a feature that benefitted anyone, i'll try adding it back in upon request :)
# the order of processing changed slightly from v1 to v2, so the output report messages vary a little
    # in v1, each genome was downloaded/parsed and scanned for the target genes in one pass
    # in v2, all input genomes are prepared for each input source, then all are searched together
# there are no longer separate summary tables for each input-genome source (accessions, fasta files, genbank files, and amino-acid files)
    # they are all represented in the primary "genomes-summary-info.tsv" table, which includes a "source" column
# references are slimmed down and hosted on GToTree github as assets now, this will help users who had trouble accessing ftp or other sites due to firewalls or other restrictions
